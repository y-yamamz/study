## 4. API設計

### 4-1. 共通仕様

| 項目             | 内容                                                  |
|------------------|-------------------------------------------------------|
| ベースURL         | `http://localhost:8080`                               |
| 通信プロトコル   | HTTP/REST                                             |
| データ形式       | JSON（Content-Type: application/json）                |
| 認証方式         | JWT Bearer トークン（Authorizationヘッダー）          |
| 認証スキップ     | `POST /api/userLogin` のみ認証不要                    |
| CORS             | `http://localhost:5173` のみ許可                     |
| エラーレスポンス | `{ "error": "メッセージ" }`（HTTP 401）              |

### 4-2. 認証系API

#### POST /api/userLogin（認証不要）

**リクエスト**
```json
{
  "userId": "admin",
  "password": "password"
}
```

**レスポンス（200 OK）**
```json
{
  "accessToken": "eyJ...",
  "refreshToken": "eyJ...",
  "userId": "admin",
  "userName": "管理者",
  "roleCd": "ADMIN"
}
```

**エラーレスポンス（401）**
```json
{ "error": "ユーザーIDまたはパスワードが正しくありません" }
```

**処理内容**
1. LoginRequest（userId, password）を受け取る
2. M_USERからuserIdでユーザー検索（YUKO_FLAG='1'の有効ユーザーのみ）
3. bcryptでパスワード検証
4. 成功時: JWTアクセストークン（1時間）＋リフレッシュトークン（7日）を生成
5. T_REFRESH_TOKENにリフレッシュトークン情報（tokenHash, revokedFlag, expiresAt, userAgent, ipAddress）を保存

### 4-3. タスク系API

#### POST /api/todoList（認証必要）

タスク一覧をシステム・プロジェクト・M_CODEをJOINして取得する。

**レスポンス（200 OK）**
```json
[
  {
    "systemCd": "001",
    "systemName": "FRSシステム",
    "projectCd": "001",
    "projectName": "FRS-AG-SYS",
    "ticketNo": "00000001",
    "revisionNo": "rev1.0",
    "statusCd": "002",
    "statusNm": "作業中",
    "deployCd": "001",
    "deployNm": "未",
    "note": "チケット内容",
    "biko": "備考",
    "yukoFlag": "1",
    "insDate": "2026-01-01T00:00:00.000",
    "insUserId": "admin",
    "updDate": "2026-03-30T00:00:00.000",
    "updUserId": "admin"
  }
]
```

**SQL概要**（TodoListMapper.xml: selectAllTodoList）
```sql
SELECT a.SYSTEM_CD, b.SYSTEM_NAME, a.PROJECT_CD, c.PROJECT_NAME,
       a.TICKET_NO, a.REVISION_NO,
       a.STATUS_CD, d.CD_NAME as STATUS_NM,
       a.DEPLOY_CD, e.CD_NAME as DEPLOY_NM,
       a.NOTE, a.BIKO, a.YUKO_FLAG
FROM TODO_LIST a
  INNER JOIN M_SYSTEM b ON a.SYSTEM_CD = b.CD
  INNER JOIN M_PROJECT c ON a.SYSTEM_CD = c.SYSTEM_CD AND a.PROJECT_CD = c.CD
  LEFT JOIN M_CODE d ON a.STATUS_CD = d.CD AND d.GR_CD = '001'
  LEFT JOIN M_CODE e ON a.DEPLOY_CD = e.CD AND e.GR_CD = '002'
ORDER BY a.SYSTEM_CD, a.PROJECT_CD, a.TICKET_NO
```

#### POST /api/todoRegister（認証必要）

タスクの一括登録・更新。新規/既存をPK（SYSTEM_CD+PROJECT_CD+TICKET_NO）で判別し、INSERT または UPDATE を実行する。

#### POST /api/todoDelete（認証必要）

タスクの削除。PK（SYSTEM_CD+PROJECT_CD+TICKET_NO）を指定して削除する。

### 9-2. フロントエンドエラー処理

|-----------------------|--------------------------------------------------------|

| APIエラー              | エラーダイアログまたはスナックバーで通知               |
| 未保存変更あり         | 「未保存の変更があります」を画面上に表示               |
| 入力バリデーションエラー| 各フィールド下部にエラーメッセージ表示                 |

### 9-3. バックエンドエラー処理

| シナリオ           | 処理方針                                            |
|-------------------|----------------------------------------------------|
| 認証失敗           | HTTP 401 + `{"error": "..."}` を返却               |
| DB接続エラー       | HTTP 500 + エラーログ出力                          |
| バリデーションエラー | HTTP 400 + フィールドごとのエラー詳細を返却       |
